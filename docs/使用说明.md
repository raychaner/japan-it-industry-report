# 📖 Docsify 网站使用说明

## 🚀 快速启动

### 方法1：使用 docsify-cli（推荐）

```bash
# 1. 安装 docsify-cli（仅需一次）
npm i docsify-cli -g

# 2. 启动本地服务器
docsify serve docs

# 3. 打开浏览器访问
# http://localhost:3000
```

### 方法2：使用 Python

```bash
# 进入 docs 目录
cd docs

# 启动 HTTP 服务器
python -m http.server 3000

# 打开浏览器访问
# http://localhost:3000
```

### 方法3：使用 VS Code Live Server

1. 安装 VS Code 扩展：**Live Server**
2. 右键点击 `docs/index.html`
3. 选择 **Open with Live Server**
4. 自动在浏览器中打开

## 🎨 自定义配置

### 修改主题颜色

编辑 `index.html` 中的 CSS 变量：

```css
:root {
  --theme-color: #42a5f5;  /* 主题色 */
  --theme-color-dark: #1976d2;  /* 深色主题色 */
}
```

### 修改网站标题

编辑 `index.html` 中的配置：

```javascript
window.$docsify = {
  name: '日本IT行业现状和趋势',  // 修改这里
  // ...
}
```

## 🌐 部署到 GitHub Pages

### 步骤1：创建 GitHub 仓库

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Add Docsify documentation"

# 关联远程仓库
git remote add origin https://github.com/你的用户名/仓库名.git

# 推送
git push -u origin main
```

### 步骤2：启用 GitHub Pages

1. 进入 GitHub 仓库页面
2. 点击 **Settings**
3. 左侧菜单选择 **Pages**
4. **Source** 选择 `main` 分支
5. **Folder** 选择 `/docs`
6. 点击 **Save**

### 步骤3：访问网站

几分钟后，您的网站将在以下地址可用：

```
https://你的用户名.github.io/仓库名/
```

## 🔧 常见问题

### Q1: .nojekyll 文件的作用是什么？

**说明**：
- `.nojekyll` 是一个空文件，位于 `docs/` 目录中
- 它告诉 GitHub Pages **不要使用 Jekyll** 处理文件
- 对于 Docsify 项目来说，这个文件是**必需的**
- 如果没有这个文件，GitHub Pages 会忽略以下划线开头的文件和目录
- **请勿删除此文件**，否则网站可能无法正常工作

### Q2: Mermaid 图表不显示？

**解决方案**：
- 确保使用 HTTP 服务器访问（不能直接双击 HTML 文件）
- 检查浏览器控制台是否有错误
- 尝试刷新页面（Ctrl+F5）

### Q3: 搜索功能不工作？

**解决方案**：
- 搜索功能需要通过 HTTP 服务器访问
- 不能直接打开 HTML 文件

### Q4: 样式显示异常？

**解决方案**：
- 检查网络连接（需要加载 CDN 资源）
- 清除浏览器缓存
- 尝试使用其他浏览器
